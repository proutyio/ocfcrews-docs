# Products

The Products collection is generated by the `@payloadcms/plugin-ecommerce` plugin and customized with the `ProductsCollection` override defined in `src/collections/Products/index.ts`.

## Product Fields

Products are organized into tabs in the admin panel:

### Content Tab

| Field | Type | Description |
|-------|------|-------------|
| `title` | text | Product name (required, max 200 chars) |
| `description` | richText | Full product description with headings, formatting, and horizontal rules |
| `gallery` | array | Product images, each with an optional variant option association |
| `layout` | blocks | Additional content blocks (CallToAction, Content, MediaBlock) |

### Product Details Tab

This tab contains the ecommerce plugin's default fields, which include:

| Field | Type | Description |
|-------|------|-------------|
| `enableVariants` | checkbox | Whether this product has variants |
| `priceInUSD` | number | Base price in USD (used when variants are disabled) |
| `variantTypes` | relationship | References to variant type definitions (e.g., "Size") |
| `variants` | array | Individual variant combinations with their own pricing |
| `inventory` | number | Stock tracking |
| `relatedProducts` | relationship (products, hasMany) | Cross-sell recommendations (excludes self) |

### SEO Tab

SEO fields powered by `@payloadcms/plugin-seo`:

| Field | Description |
|-------|-------------|
| Meta Title | Page title for search engines |
| Meta Description | Page description for search results |
| Meta Image | Social sharing image |
| Overview | SEO quality overview |
| Preview | Live preview of search result appearance |

### Sidebar Fields

| Field | Type | Description |
|-------|------|-------------|
| `categories` | relationship (categories, hasMany) | Product categories for filtering |
| `crew` | relationship (crews) | If set, restricts product visibility to members of this crew |
| `slug` | text | URL-friendly identifier (auto-generated by plugin) |

## Variants

When `enableVariants` is checked, a product can have multiple variants (e.g., different sizes or colors). Each variant can have:

- Its own **price** (`priceInUSD` on the variant)
- Associated **gallery images** (matched via `variantOption` on gallery items)
- Specific **variant options** (e.g., "Large", "Blue")

### Variant-Specific Gallery Images

Gallery images can be associated with specific variant options. When a customer selects a variant, the product page shows the matching gallery image:

```typescript
const imageVariant = product.gallery?.find((galleryItem) => {
  const variantOptionID = typeof galleryItem.variantOption === 'object'
    ? galleryItem.variantOption.id
    : galleryItem.variantOption
  return variant?.options?.some((option) => {
    const optionID = typeof option === 'object' ? option.id : option
    return optionID === variantOptionID
  })
})
```

## Crew-Scoped Products

Products can optionally be associated with a specific crew via the `crew` field. When set, this restricts who can see and purchase the product to members of that crew (those with coordinator, leader, or member roles).

## Access Control

### Create

```
admin, crew_coordinator
```

Crew coordinators can create products, which will be automatically scoped to their crew via a `beforeChange` hook.

### Read

Products use the plugin's default read access, which typically allows reading published products.

### Update

| Role | Access |
|------|--------|
| admin | All products |
| crew_coordinator | Products where `crew` equals their crew ID |

### Delete

| Role | Access |
|------|--------|
| admin | All products |
| crew_coordinator | Products where `crew` equals their crew ID |

### Crew Enforcement Hook

A `beforeChange` hook ensures coordinators can only assign products to their own crew:

```typescript
({ data, req }) => {
  if (!checkRole(['admin'], req.user) && checkRole(['crew_coordinator'], req.user)) {
    data.crew = getCrewId(req.user) ?? data.crew
  }
  return data
}
```

## Admin Configuration

- **Group**: Shop
- **Default columns**: title, enableVariants, _status, variants.variants
- **Title field**: title
- **Live Preview**: Supported via `generatePreviewPath`

## Default Populate

For performance, the default populate configuration selects only the essential fields:

```typescript
defaultPopulate: {
  title: true,
  slug: true,
  variantOptions: true,
  variants: true,
  enableVariants: true,
  gallery: true,
  priceInUSD: true,
  inventory: true,
  meta: true,
}
```
