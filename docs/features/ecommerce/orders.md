# Orders

Orders are created when a customer successfully completes the checkout process. The Orders collection is generated by the `@payloadcms/plugin-ecommerce` plugin and customized with access control overrides.

## Order Lifecycle

```
Cart -> Checkout -> Payment (Stripe) -> Order Created -> Order Viewable
```

1. **Cart Phase**: Items accumulate in the user's cart as they browse and add products.
2. **Checkout Phase**: The user provides contact information, addresses, and initiates payment.
3. **Payment Phase**: Stripe processes the payment via PaymentIntent.
4. **Order Creation**: Upon successful payment confirmation (`confirmOrder`), the ecommerce plugin creates an order record with all the relevant details.
5. **Viewable**: The customer is redirected to the order detail page.

## Order Data

Orders are created by the ecommerce plugin and typically include:

| Field | Description |
|-------|-------------|
| `customer` | Reference to the user who placed the order |
| `items` | Array of purchased products with quantities and variant selections |
| `total` | Order total amount |
| `currency` | Currency code (USD) |
| `status` | Order status: `processing`, `shipped`, `completed`, `cancelled`, `refunded` |
| `trackingNumber` | Shipping tracking number (shown to customer when status is `shipped`) |
| `crews` | Relationship (hasMany) to crews involved in the order. Auto-populated from order items' product crews via `afterChange` hook. Read-only. |
| `stripeTransfers` | JSON field storing Stripe Connect transfer records (admin-only visibility, read-only) |
| `createdAt` | Timestamp when the order was placed |

## Access Control

**Source**: `src/plugins/index.ts`

The orders collection has customized access control:

### Read Access

```typescript
read: ({ req: { user } }) => {
  if (!user) return false
  if (checkRole(['admin'], user)) return true
  if (checkRole(['crew_coordinator', 'shop_admin'], user)) {
    const crewId = getUserCrewId(user)
    if (crewId) {
      return {
        or: [
          { customer: { equals: user.id } },
          { crews: { contains: crewId } },
        ],
      }
    }
  }
  return { customer: { equals: user.id } }
}
```

| Role | Access |
|------|--------|
| admin | All orders |
| crew_coordinator, shop_admin | Their own orders OR orders containing products from their crew (via `crews` field) |
| Authenticated user | Only their own orders (via `Where` constraint on `customer`) |
| Unauthenticated | No access |

This ensures customers can only view their own order history, coordinators and shop admins can see orders involving their crew's products, and admins have full visibility across all orders.

### Other Operations

Create, update, and delete operations use the plugin's default access controls.

## Customer View

### Order Detail Page

After completing checkout, customers are redirected to `/shop/order/{orderID}`. This page displays:

- Order confirmation details
- Items purchased with images and variant information
- Order total
- Timestamps

### Account Integration

Orders are also accessible through the user's account. The Users collection includes a `join` field that connects to orders:

```typescript
{
  name: 'orders',
  type: 'join',
  collection: 'orders',
  on: 'customer',
  admin: {
    allowCreate: false,
    defaultColumns: ['id', 'createdAt', 'total', 'currency', 'items'],
  },
}
```

This provides an orders tab on each user's admin page, showing their purchase history.

## Admin View

In the Payload admin panel, orders are grouped under the **Shop** section. Admins can:

- View all orders across all customers
- Filter and search orders
- View order details including customer information, items, and payment status

## Order Status

Order status tracking is managed by the ecommerce plugin. The status reflects the current state of the order as it progresses through fulfillment. Available statuses: `processing`, `shipped`, `completed`, `cancelled`, `refunded`.

## Hooks

### `beforeChange`

- **Checkout gate**: On create, checks the global `settings.checkoutDisabled` flag and per-crew disablement. If checkout is disabled for the user's crew, throws an error preventing order creation.

### `afterChange`

Three `afterChange` hooks run on order mutations:

1. **Crew auto-population** (create only): Iterates over order items, looks up each product's crew, and writes the unique set of crew IDs to the `crews` relationship field. This enables crew-scoped order visibility for coordinators and shop admins.

2. **Order confirmation email** (create only): Sends an email to the customer with order details (items, total, order URL) using the `OrderConfirmationEmail` template via Resend.

3. **Status update email** (update, when status changes): Sends an `OrderStatusUpdateEmail` to the customer with the new status and tracking number (if shipped). On `refunded` status, also reverses Stripe Connect transfers for crew payouts.
