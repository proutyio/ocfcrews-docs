# Pages

## Overview

The **Pages** collection stores CMS-managed website pages with a flexible block-based layout system. Pages support a hero section, multiple content blocks, SEO metadata, draft/publish workflow, live preview, and URL slugs. This is the primary collection for building the public-facing website.

## Configuration

| Property | Value |
|---|---|
| **Slug** | `pages` |
| **Admin Group** | Content |
| **Use As Title** | `title` |
| **Default Columns** | `title`, `slug`, `updatedAt` |
| **Live Preview** | Enabled (generates preview URL from slug) |
| **Versions** | Drafts with autosave enabled, max 50 versions per document |

## Fields

### Top-Level Fields

| Name | Type | Required | Description |
|---|---|---|---|
| `title` | `text` | Yes | The page title (max 200 characters). |
| `publishedOn` | `date` | No | Publication date with day-and-time picker. Auto-set to the current date when status changes to "published" if not already set. Displayed in the sidebar. |
| `slug` | `text` | No | URL-friendly slug, auto-generated by Payload's built-in `slugField()`. |

### Hero Tab

The hero section is a group field with the following sub-fields:

| Name | Type | Required | Description |
|---|---|---|---|
| `hero.type` | `select` | Yes | Hero display type. Options: `none`, `highImpact`, `mediumImpact`, `lowImpact` (default). |
| `hero.richText` | `richText` | No | Hero text content with Lexical editor (supports h1-h4, fixed and inline toolbars). |
| `hero.links` | `array` | No | Up to 2 link items for the hero section. |
| `hero.media` | `upload` (media) | Yes | Hero image. Only shown when type is `highImpact` or `mediumImpact`. |

### Content Tab

| Name | Type | Required | Description |
|---|---|---|---|
| `layout` | `blocks` | Yes | The page body composed of content blocks. |

Available block types:

- **CallToAction** -- Promotional call-to-action section
- **Content** -- Rich text content section
- **MediaBlock** -- Media display block
- **Archive** -- Archive/listing block
- **Carousel** -- Image or content carousel
- **ThreeItemGrid** -- Three-item grid layout
- **Banner** -- Banner/alert block
- **FormBlock** -- Embedded form block

### SEO Tab (`meta`)

| Name | Type | Required | Description |
|---|---|---|---|
| `meta.title` | `text` | No | SEO meta title (supports auto-generation). |
| `meta.image` | `upload` (media) | No | SEO open graph image. |
| `meta.description` | `textarea` | No | SEO meta description. |
| Overview | `ui` | -- | SEO overview panel showing title, description, and image status. |
| Preview | `ui` | -- | Search engine result preview (supports auto-generation). |

## Access Control

| Operation | admin | editor | viewer | Unauthenticated |
|---|---|---|---|---|
| **Create** | Yes | No | No | No |
| **Read** | Full access | Full access | Full access | Published pages only (`_status = published`) |
| **Update** | Yes | Yes | No | No |
| **Delete** | Yes | Yes | No | No |

- **`adminOnly`** -- Only users with the `admin` role can create pages.
- **`adminOrEditor`** -- Users with `admin` or `editor` roles can update and delete pages.
- **`adminOrPublishedStatus`** -- Admin, editor, and viewer roles see all pages; unauthenticated users and other roles only see published pages.

## Hooks

### `afterChange` -- `revalidatePage`

Triggers Next.js revalidation when a page is changed:
- If the page is published, revalidates both the page path (e.g., `/about` or `/` for the home page) and the `pages` cache tag.
- If the page was previously published but is now unpublished, revalidates the old path to remove it from the cache.
- Respects the `disableRevalidate` context flag.

### `afterDelete` -- `revalidateDelete`

Revalidates the deleted page's path and the `pages` cache tag to ensure the page is removed from the Next.js cache. Respects the `disableRevalidate` context flag.

### Field Hook: `publishedOn.beforeChange`

Auto-populates the `publishedOn` date with the current timestamp when a page's status changes to "published" and no date has been explicitly set.
