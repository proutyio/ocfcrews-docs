Polling Impact
The cost of removing Ably
Without Ably, every user with a chat tab open generates continuous HTTP polling requests. Each poll invokes a Vercel serverless function and runs 1–2 database queries. The load scales linearly with active chat users.

Total polling load ~24 q/s ~57 q/s
With Ably: Chat polling load is ~0 q/s (writes only). Only notification polling remains at ~0.4 q/s per 50 users.
Without Ably: ~24 extra queries/sec per 50 active chat users. At 100 chat users, that's 57 q/s of pure polling overhead.
Vercel invocations: 50 chat users × (6 + 2 + 20) polls/min = 1,400 invocations/min = ~2M/month of additional serverless calls.

Bottom Line

Without Ably, the system works but chat becomes the dominant source of server load. HTTP polling at 10s intervals generates ~24 queries/sec per 50 chat users and ~2M Vercel invocations/month. Comfortable capacity drops from ~500 to ~300 concurrent users, and chat latency rises from ~50ms to 5–10 seconds.

You save $25/mo on Ably but spend $8–15/mo more on Vercel invocations, making the real savings $10–17/mo — in exchange for significantly worse chat UX and ~40% less headroom. At scale, the cost difference narrows further as polling overhead compounds. Ably pays for itself above ~100 users.
